이 페이지는 6v6-web (Apache + ModSecurity) 가 응답한 랜딩 페이지입니다. 모든 취약 사이트 트래픽은 web 의 ModSecurity 검사를 통과한 후 백엔드 컨테이너로 전달됩니다.

보안장비 교육 콘솔 (secuops-easy 특강)

방화벽·IPS·WAF 를 장비형 웹 콘솔로 직접 다루는 입문 특강. 폼 조작이 만드는 실제 명령(nft / Suricata rule / SecRule)을 미리보기로 보여준 뒤 적용한다. (fw 의 HAProxy 가 Host 헤더로 라우팅 — 위 hosts 한 줄에 세 이름 포함됨)

교육방화벽 콘솔 (nftables)fw-gui.6v6.lab — 룰 / NAT / conntrack / 로그 / SIEM + 침해대응 시나리오 10 교육IPS 콘솔 (Suricata)ips-gui.6v6.lab — 탐지룰 구조·작성 / eve.json / SIEM + 침해대응 시나리오 30 교육WAF 콘솔 (ModSecurity)waf-gui.6v6.lab — SecRule 작성 / CRS / audit 분석 / SIEM + 침해대응 시나리오 30

교안·실습(6주)은 CCC 6v6 Training 메뉴의 secuops-easy 과목. 콘솔의 모든 동작은 적용 전 미리보기/문법검사를 거쳐 안전하게 연습할 수 있다.

학습 포인트: Wazuh agent 는 자체 binary 가 로그 수집 + 디코딩 + 알림 관리자에게 보낸다 (TCP/1514, 암호화). syslog forward 는 rsyslog 가 모든 로그를 그대로 UDP/514 로 전달하고, manager 가 직접 디코딩한다. agent = 정밀 + 보안적으로 우수, syslog = 단순 + 표준 호환.

네트워크 구성 (4-tier: ext → pipe → dmz → int, + user/wan)
