이 방화벽은 무엇을 하나요?

방화벽(nftables) 은 el34 의 가장 바깥 관문입니다. 외부(ext)에서 들어온 트래픽은 이 fw 를 거쳐야 내부로 갈 수 있습니다. 이 콘솔에서 만드는 모든 동작은 실제 nft 명령 으로 변환되어 적용됩니다 — 버튼이 아니라 명령을 배웁니다.

  • input — 방화벽 자신에게 오는 트래픽
  • forward — 방화벽을 통과해 내부로 가는 트래픽
  • output — 방화벽에서 나가는 트래픽
네트워크 인터페이스 · 존(zone)

fw 는 두 개의 다리를 가집니다. eth0 은 외부(ext)로, eth1 은 ips 로 가는 사이망(pipe)으로 향합니다.

객체(그룹·Alias) 만들기 여러 IP/포트를 묶어 룰에서 @이름 으로 재사용

상용 방화벽의 주소 객체 / 주소 그룹 / Alias 에 해당한다. nftables 에서는 named set 으로 만든다. 여러 악성 IP 를 한 그룹으로 묶어두면, 룰마다 IP 를 나열하지 않고 @그룹이름 한 번으로 참조한다. 그룹에 IP 를 추가/삭제하면 그 그룹을 쓰는 모든 룰에 즉시 반영된다.

여기에 생성될 nft add set / nft add element 명령이 표시됩니다.

만든 그룹은 룰 관리 의 출발지/목적지에 @그룹이름 으로 사용 (예: 출발지 = @blocklist).

연결 추적 (conntrack) —

방화벽은 패킷을 하나씩 보지 않고 연결(흐름) 단위로 기억합니다. 한 번 허용된 연결의 응답 패킷은 ct state established,related 로 자동 통과합니다. 이것이 stateful inspection 입니다.

방화벽 이벤트(적용/삭제/리셋)는 /var/log/nft_edu/events.log 에 JSON 으로 남습니다. 연동을 켜면 fw 의 Wazuh 에이전트가 이 파일을 감시(tail)하여 SIEM 매니저(10.20.32.100)로 전송하고, Wazuh 대시보드에서 방화벽 활동을 볼 수 있습니다.
