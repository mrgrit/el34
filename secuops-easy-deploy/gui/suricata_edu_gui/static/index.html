IPS(Suricata)는 무엇을 하나요?

IPS/IDS(Suricata) 는 방화벽을 통과한 트래픽의 내용(payload) 을 검사한다. 방화벽이 "누가/어디로"(IP·포트)를 본다면, IPS 는 "무엇을 담고 있나"(공격 패턴)를 본다. el34 에서 Suricata 는 fw↔ips 사이망(eth0)을 감시하며, 탐지 결과를 eve.json 에 기록한다.

  • 탐지룰(rule) — action·프로토콜·방향 헤더 + 옵션(content/pcre/flow/sid)
  • eve.json — 모든 이벤트(alert/http/dns/tls/flow)의 JSON 로그
  • 이 콘솔의 룰은 만들 때마다 실제 Suricata rule 문법 으로 변환되어 reload 된다.

SIEM (Wazuh) 연동

Suricata 의 eve.json 은 ips 의 Wazuh 에이전트가 직접 tail 하여 SIEM 매니저(10.20.32.100)로 보낸다. 대부분의 el34 환경에서는 이미 연동되어 있다 — alert 가 Wazuh 대시보드에 rule.id 86601 등으로 나타난다.
